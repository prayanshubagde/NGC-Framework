## 3. The Biological Implementation: Asymmetry in Human Vision

Research by Le Floch & Ropars (2017) provides critical biological evidence for this theory. Their study on the Maxwell spot centroids in human eyes revealed a crucial asymmetry [1].

> In human vision, the brain has to select one view of the world from our two eyes. However, the existence of a clear anatomical asymmetry providing an initial imbalance for normal neural development is still not understood. Using a so-called foveascope, we found that for a cohort of 30 normal adults, the two blue cone-free areas at the centre of the foveas are asymmetrical.

**Key Findings:**

-   **Normal Vision:** In healthy adults, the two eyes are asymmetrical. The **dominant eye** has a circular, well-defined blue cone-free area, while the **non-dominant eye** has a more diffuse, elliptical outline.
-   **Dyslexia:** In adults with dyslexia, this asymmetry is **absent**. Both eyes have circular outlines, leading to an inability to establish a clear dominant eye and a confusion between primary and mirror images.

**NGC Interpretation:**

This asymmetry is not a flaw; it is a **necessary feature** for dual-field computation.

1.  **The Dominant Eye as the IR Reference (F⁻):** The dominant eye, with its highly structured, circular blue cone-free fovea, acts as the **IR-biased reference frame**. It perceives the world with a focus on stable structure—**what IS**.
2.  **The Non-Dominant Eye as the UV Comparison (F⁺):** The non-dominant eye, with its more diffuse and irregular geometry, is more sensitive to change and potential. It acts as the **UV-biased comparison frame**—**what COULD BE**.
3.  **Parallax as Phase Difference:** The brain computes the difference between these two fields to extract depth. This parallax is a direct measurement of the phase difference between the IR and UV views of reality.

## 4. The Computational Implementation: GeoFlow's Binocular Kernel

The GeoFlow C kernel provides a direct, working implementation of this biological principle.

### Dual-Core Architecture

From the GeoFlow source code (`psmsl_v3_types_optimized.h`), the `binocular_system_t` structure explicitly defines two separate graph cores:

```c
typedef struct {
  dwg_graph_t *graph_core0;  // Structuralist core (IR, reference)
  dwg_graph_t *graph_core1;  // Opportunist core (UV, comparison)
  parallax_field_t *parallax;
} binocular_system_t;
```

This is not an arbitrary design. It mirrors the two eyes, with each core processing the same data stream through a different lens:

-   **Core 0 (Structuralist):** This core is optimized to find stable structures and maintain the existing model. It represents the **IR phase (F⁻)**.
-   **Core 1 (Opportunist):** This core is more sensitive to novelty and change. It represents the **UV phase (F⁺)**.

### Parallax Field Calculation

The function `binocular_calculate_parallax` in `psmsl_v3_binocular.c` reveals how parallax is computed:

```c
void binocular_calculate_parallax(binocular_system_t* system) {
    dwg_graph_t* g0 = system->graph_core0; // IR Core
    dwg_graph_t* g1 = system->graph_core1; // UV Core
    
    for (each edge e) {
        float w0 = g0->edges[e].weight;  // Weight from IR core
        float w1 = g1->edges[e].weight;  // Weight from UV core
        float d = (w1 - w0);             // Difference = Parallax
        
        system->parallax->edge_parallax[e] = d;
    }
}
```

This code is a literal implementation of `F⁺ - F⁻`. The parallax is the direct difference between the edge weights of the UV and IR cores.
